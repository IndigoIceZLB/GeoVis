**GeoResearchVis** 是一款基于 .NET 8 开发的桌面端轻量级空间数据可视化与分析工具。

本项目主要为城市规划、交通地理等领域的科研人员提供一个开箱即用的本地化交互环境。旨在作为传统 Python (Pandas/GeoPandas) 脚本出图工作流的补充，通过内置的列式数据库引擎，在常规桌面硬件配置下，实现对较大规模空间信令数据（如 OD 轨迹、网格人口、气象降水）的高效关联查询与可视化呈现。

---

## 🔹 核心功能 (Features)

1. **本地数据库集成**
   - 采用 DuckDB 作为内嵌数据引擎，支持将结构化的 CSV 数据直接转存为本地持久化文件 (`.duckdb`)。
   - 依赖 SQL 进行底层的分组聚合与空间属性连接，减少将全量数据加载至内存的开销。
2. **多维度时序图表绘制**
   - 使用 ScottPlot 构建本地时序图表，支持通过鼠标进行平移和缩放。
   - 支持多 Y 轴数据的对齐叠加展示。例如：同一时间轴上展示 OD 总流量（折线）、边拓扑数量（折线）以及对应的气象站降水量（采用倒挂柱状图形式，避免遮挡主要折线）。
3. **基于时间切片的空间交互**
   - 采用 WebView2 + Leaflet.js 架构渲染空间底图。
   - 提供 0-23 小时时间滑块联动功能。在拖动滑块时，后台依据当前时间戳重算各网格属性（如：流出流入量、人口环比差值），并通过更新 GeoJSON 属性实现地图斑块颜色的动态刷新。
4. **完全离线运行**
   - 地图底图样式、JS 依赖库及业务数据处理均在本地运行，无需连接外部网络，适用于对数据隐私要求较高的研究场景。
5. **高度解耦的空间分析模式**
   - 实现了“渲染图层 (OD / 驻留人口)”与“分析模式 (绝对值 / 时空环比做差)”的正交解耦。
   - 支持 OD 数据的多指标视图切换（流出流量、流入流量、总流量），无需重新处理底层数据即可动态重绘地图着色与信息提示窗 (Tooltip)。
6. **多基准日期的均值做差模型**
   - 在底层的 DuckDB CTE (公共表表达式) 中实现了多日期的均值聚合逻辑。
   - 用户可勾选多个历史日期作为对比基线，引擎将在后台实时计算这些日期同一时间段的网格人口/流量平均值，并与当前观察日期进行差值计算与冷暖色渲染。
7. **融合 ECharts 与图表锁定交互**
   - **微观气象悬浮窗**：在 WebView2 内部集成了 Apache ECharts 渲染引擎。克服了原生 WPF 的空域遮挡问题 (Airspace Issue)，实现了支持自由拖拽缩放、一键全选过滤的右下角降水数据悬浮面板。
   - **时序图表 Y 轴锁定**：引入 ScottPlot 5 的 `AxisRules` 特性，支持一键锁定坐标轴的 Y 轴缩放比例，便于用户在长序列横向平移时保持稳定的数据对比参照系。
8. **视口状态保持与响应式 UI 架构**
   - **视觉中心锚定**：在进行指标切换或叠加不同气象站数据时，图表引擎能自动捕捉并恢复上一帧的 X 轴缩放级别与平移位置。
   - **自适应空间布局**：左侧控制面板引入原生 WPF Expander 与滚动机制，在小尺寸屏幕下防止庞大的参数配置区发生遮挡。
9. **Top 10 辐射流向可视化 (Flow Map)**
   - 在网格多边形上方叠加渲染矢量飞线，展示绝对值最高或环比差值最大的 Top 10 OD 流量。
   - 支持图表联动与交互下钻：通过点击地图上的特定网格，动态过滤并渲染以此网格为中心的“流入/流出”局部辐射流向；提供剔除自流 (O=D) 与激增/锐减趋势过滤功能。

## 🏗️ 技术栈 (Tech Stack)

- **前端界面 (UI):** WPF (.NET 8), CommunityToolkit.Mvvm
- **数据访问 (Data Access):** DuckDB.NET, Dapper
- **统计图表 (Charts):** ScottPlot 5
- **空间渲染 (Web Map):** Microsoft.Web.WebView2, Leaflet.js

## 📊 数据输入规范 (Data Schema)

使用本软件前，需将原始数据预处理为以下标准格式的 CSV 文件和 GeoJSON 空间文件：

1. **OD 轨迹流长表 (`od_data`)**: 
   包含字段 `start_date`, `start_hour`, `end_hour`, `o_grid` (起点网格ID), `d_grid` (终点网格ID), `trip_cnt` (出行量)。
2. **网格驻留信号密度表 (`mobility_data`)**: 
   包含字段 `date`, `hour`, `grid_id`, `signal_count`。
3. **网格常住人口表 (`pop_data`)**: 
   包含字段 `grid_id`, `real_pop_sum`, `unicom_cnt`。
4. **小时气象降水极简表 (`rainfall_data`)**: 
   包含字段 `timestamp`, `station`, `precip_mm`。
5. **空间网格边界 (`.geojson`)**: 
   采用 WGS84 (EPSG:4326) 坐标系，`properties` 属性块中必须包含主键 `cid`，用于与上述 CSV 中的 `grid_id` 等字段进行空间连接 (Join)。

## 🚀 使用指南 (Getting Started)

### 1. 编译与运行
- 建议使用 **Visual Studio 2022** 及以上版本打开 `GeoResearchVis.sln`。
- 确保已安装 `.NET 8.0 桌面开发` 工作负载。
- 确认项目 `Assets/web/map_template.html` 文件的属性已设置为 **“生成操作: 内容”** 且 **“复制到输出目录: 如果较新则复制”**。
- 生成解决方案并运行。

### 2. 操作流程

**步骤一：数据导入**
- 在软件左侧面板的“数据建库”区域，根据按钮提示依次选择上述准备好的 CSV 文件。
- 数据将转换为同目录下的 DuckDB 文件。如需更新数据，可点击“清空”按钮后重新导入。

**步骤二：时序特征观察**
- 切换至 **“网络与气象时序统计”** 选项卡。
- 通过顶部下拉框，可切换需要观察的分析指标（如 OD 流量或驻留人口）。
- 若已导入降水数据，可在右侧下拉框选择特定气象站点（或选择全部站点叠加），观察降水对流量变化的干预情况。

**步骤三：空间底图呈现**
- 在左侧面板点击 **“加载网格(GeoJSON)渲染”**，选择准备好的网格底图文件。
- 切换至 **“空间底图呈现”** 选项卡。
- 使用左下角的**时间切片控制台**，调整日期和小时，观察地图上对应网格指标（如冷暖色表示的人口增减差异）的空间分布特征。

## 📝 注意事项

- **图表标签语言**：为保证在不同 Windows 系统的底层绘图引擎（SkiaSharp）下字体渲染的稳定性，图表标签、坐标轴默认采用全英文字符与数值显示。
- **性能预期**：软件的数据处理速度受限于运行设备的 CPU 性能与固态硬盘 I/O 速度。在主流消费级 PC 上，处理常规城市级月度信令数据时，视图切换与数据过滤均能保持较好的流畅度。
