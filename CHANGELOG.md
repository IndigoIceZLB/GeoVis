# 更新日志 (Changelog)
## [v1.0.0] - 2026-03-31
- **Publish**：支持空间驻留数据、OD数据以及降水数据综合查询功能。

## [v1.1.0] - 2026-04-01
### 优化 (Enhancements)
- **图表自动刷新逻辑**：修复了在网络指标与驻留指标间切换时图表未能自动刷新的 Bug。
- **图表视口状态保持**：在切换数据指标或叠加不同气象站时，ScottPlot 图表引擎会自动记录并恢复当前的 X 轴平移与缩放级别，保持长时序观察的连贯性。
- **响应式 UI 布局**：左侧控制面板按业务逻辑重构为可折叠模块（WPF Expander），并引入全局滚动条，解决小尺寸屏幕或窗口调整时控制项被遮挡的问题。
### 新增功能 (New Features)
- **Top 10 OD 飞线可视化**：在空间底图上新增带透明度的矢量虚线图层，支持展示全图或特定网格的 Top 10 OD 流量关系。
- **微观辐射分析交互**：支持在地图上点击特定网格，飞线会自动收束，展示以此网格为中心的流入/流出 Top 10 关系；点击空白处恢复全局视野。
- **精细化飞线控制**：
  - 提供“剔除网格内自流 (O=D)”开关，过滤对角线效应数据，突出跨区流动。
  - 在“时空环比做差”模式下，提供飞线趋势过滤：支持选择“仅看激增 (红线)”、“仅看锐减 (蓝线)”或“变化绝对值最大”。
### 修复 (Bug Fixes)
- 修复了在网络指标与驻留指标间切换时，图表未能自动刷新的问题。
- 修复了在 DuckDB 做差模式下，剔除自流时因 `FULL OUTER JOIN` 别名改变导致的 `Binder Error` SQL 语法错误。
- 修复了 WebView2 解析前端传回的纯数字网格 ID 时产生的隐式类型转换异常，解决了点击网格无响应的问题。

## [v1.2.0] - 2026-04-10
### 优化 (Enhancements)
- **统计计算下推**：重构了多站点气象数据的统计逻辑。将包络带的极值 (Min/Max) 与均值 (Avg) 计算完全下推至 DuckDB 引擎，消除了 C# 内存聚合导致的“数据漂移”现象，并显著降低了内存占用。
- **透明度控制机制**：网格透明度调整操作由后端数据重载重构为前端 CSS `fillOpacity` 直接控制，消除了 IPC (进程间通信) 开销，实现了无延迟的实时视觉反馈。
- **地图服务解耦**：采用底层 XYZ 瓦片接口替换了原有的 JS SDK 调用方式，移除了对外部 API Key 的依赖，提升了离线与内网环境下的兼容性。

### 新增功能 (New Features)
- **多底图动态切换**：空间视图新增底图源热切换功能，内置 Carto 极简学术白、高德标准街道图及高德高清卫星图。
- **动态坐标系纠偏**：底层渲染管线内置 WGS-84 至 GCJ-02 转换算法。在切换至高德底图时，GeoJSON 网格与 OD 飞线能够自动执行前端动态纠偏，确保空间对齐精度。
- **气温时序与包络带分析**：
  - 新增小时气温数据 (`temperature_data`) 的入库与时序折线展示。
  - 在全选模式下，采用“均值主线 + 极值透明包络带 (Min-Max Envelope Band)”替代传统多折线堆叠图，直观反映全局气温中枢与空间离散程度。

### 修复 (Bug Fixes)
- 修复了 ScottPlot 5 在处理带 Alpha 通道的十六进制颜色字符串时发生的透明度丢失问题。
